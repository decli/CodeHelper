# 取件码助手（CodeHelper）

一款为老人场景设计的本地短信取件码助手。

## 功能

- 首屏默认读取最近 12 小时短信，只展示未取件码
- 首页按原型图拆分为 `时间筛选`、`强制展示所有 / 设置`、`未取件统计` 三块区域，适合老人直接查看
- 时间筛选默认选中 `12小时`，点击 `其它时间` 会弹出更大字号的时间选择面板，可直接选择 `1天内`、`2天内`、`3天内`、`5天内`、`7天内`、`14天内`，选择后立即重新读取短信
- 当前时间范围会扫描本机短信表中的全部短信记录，不再只依赖单一 `Inbox` 视图，减少同一发件人历史取件码漏读
- 同时兼容 `SMS + MMS` 两类本机接收消息，减少“菜鸟驿站”这类平台消息在系统短信列表可见、但应用内漏读的情况
- 仅展示本机收到的取件码，不展示本机发出的短信内容
- `未取件码` 统计会按未取件的实际取件码条数累加，同一短信里有 3 个取件码就计 3 个
- 点击 `强制展示所有` 会重新加载当前时间范围内的全部取件码，同时展示未取件与已取件
- 未取件卡片支持左滑抽屉式展开 `已取件` 按钮，点击后立即标记为已取件并从默认列表消失
- 已取件卡片在 `强制展示所有` 结果中始终排在未取件之后；未取件按短信时间倒序排列
- 同一条短信里的多个取件码会聚合到一张卡片里，卡片内按行展示，每个取件码一行
- 默认提取逻辑会在 `取件码`、`提货码`、`货码`、`驿站码`、`凭` 等提示词后面，自动识别一个或多个由字母、数字、连字符组成的取件码，兼容 `取件码为`、`凭“1-3-27017”`、`货码为3-3-06006，8-5-06003` 等写法
- 设置页支持用普通提示词扩展识别范围，只有遇到特殊平台时才需要补充高级正则规则；内置 `恢复默认设置`，提示词为空或高级规则语法错误时不允许保存
- 重新设计了更符合“短信取件码助手”语义的应用图标
- 全程离线运行，不申请网络权限

## 权限说明

- 应用核心读取能力以 `v1.0.12` 代码版本为准，不再要求用户将应用设置为默认短信应用
- 部分厂商系统会将物流平台短信归类为“通知类短信”或类似受保护通道。以小米系统为例，除普通短信读取授权外，还需要在系统设置中额外开放此类通知短信访问权限，否则系统短信列表里可见的物流消息仍可能无法被应用读取
- 如果复制同一条短信内容后由本机号码重新发送，应用可以识别，而原始平台短信无法识别，通常就是厂商短信权限分流导致，不是取件码正则规则本身的问题

## 构建

```bash
./gradlew assembleRelease
```

推送到 `main` 后，GitHub Actions 会执行构建校验并上传 APK artifact。
推送 `v*` 标签后，GitHub Actions 会在 Releases 页面创建正式版本并附带可下载 APK。
